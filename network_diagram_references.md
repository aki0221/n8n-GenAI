# ネットワーク図の参考資料と設計ガイドライン

## 収集した参考資料

### 1. 関係性ネットワークマッピングのガイド
- 出典: [Simply Stakeholders](https://simplystakeholders.com/relationship-network-mapping/)
- 主要ポイント:
  - ノード（人やグループ）と接続線で関係性を視覚化
  - 複雑な関係性を階層的に表現
  - 関係の強さや種類を線の太さや種類で区別
  - グループ化による視認性の向上

### 2. ネットワーク図のベストプラクティス
- 出典: [Lucidchart Blog](https://www.lucidchart.com/blog/network-diagramming-best-practices)
- 主要ポイント:
  - 業界標準のアイコンと命名規則の使用
  - 詳細と明確さのバランス
  - 階層的な表現（高レベル概要と詳細図の併用）
  - 接続タイプの明確な区別

### 3. ネットワーク図の基本と作成方法
- 出典: [Auvik](https://www.auvik.com/franklyit/blog/effective-network-diagrams/)
- 主要ポイント:
  - 物理層と論理層の区別
  - シンボルの標準化（円=Layer 3、四角=Layer 2など）
  - 複雑な要素のグループ化
  - 色分けによる視覚的区別

### 4. Mermaidフローチャート構文
- 出典: [Mermaid公式ドキュメント](https://mermaid.js.org/syntax/flowchart.html)
- 主要ポイント:
  - ノードの形状バリエーション（四角、丸角、円など）
  - 方向設定（TB, TD, BT, RL, LR）
  - エッジの表現方法（実線、点線、矢印など）
  - サブグラフによるグループ化

### 5. Mermaid ERダイアグラム
- 出典: [Mermaid公式ドキュメント](https://mermaid.js.org/syntax/entityRelationshipDiagram.html)
- 主要ポイント:
  - エンティティ間の関係性表現
  - カーディナリティマーカー（一対多、多対多など）
  - 識別関係と非識別関係の区別（実線と点線）
  - 属性の表示方法

## 設計ガイドライン

これらの参考資料から抽出した、パラメータ間の相互関係ネットワーク図作成のためのガイドラインは以下の通りです：

### 1. 構造とレイアウト
- **階層的構造**: パラメータを論理的なグループに分類し、階層構造で表現
- **方向性**: 上から下（TB）または左から右（LR）の一貫した方向性を維持
- **グループ化**: 関連するパラメータをサブグラフでグループ化
- **バランス**: 視覚的な複雑さと情報の詳細さのバランスを取る

### 2. ノードの表現
- **形状による区別**: 
  - 評価軸パラメータ: 四角形 `[]`
  - 重み付けパラメータ: 丸角四角形 `()`
  - 閾値パラメータ: 六角形 `{{}}`
  - 調整パラメータ: 円形 `(())`
- **色分け**:
  - テクノロジー視点: 青系
  - マーケット視点: 緑系
  - ビジネス視点: 赤系
  - メタパラメータ: 紫系

### 3. エッジ（接続線）の表現
- **関係の種類**:
  - 階層的依存関係: 実線矢印 `-->`
  - 影響関係: 点線矢印 `-..->`
  - 調整関係: 太線矢印 `==>`
  - フィードバック関係: 曲線矢印 `--o`
- **ラベル付け**: 関係の性質を簡潔に説明するラベルを追加

### 4. 視認性と理解しやすさ
- **簡潔なラベル**: 各ノードに簡潔で明確なラベルを付ける
- **適切な間隔**: ノード間に十分な間隔を確保
- **凡例の追加**: 使用した形状、色、線種の意味を説明する凡例を含める
- **注釈**: 複雑な関係や重要なポイントには注釈を追加

これらのガイドラインに基づいて、パラメータ間の相互関係を示すネットワーク図を設計・実装します。
