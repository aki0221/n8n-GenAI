# 異なる検出アルゴリズムの比較図設計

## 設計目的
コンセンサスモデルにおける異なる静止点検出アルゴリズムの特徴、長所、短所、適用シナリオを視覚的に比較し、読者が自組織のニーズに最適なアルゴリズムを選択できるようにする。

## 設計要件
1. パート4のドキュメントに記載されている異なる検出アルゴリズムを網羅
2. 各アルゴリズムの主要特性を明確に比較
3. 適用シナリオや条件による選択基準を示す
4. n8nでの実装の複雑さや考慮点を含める
5. 視覚的に分かりやすく、一目で違いが理解できる表現

## 比較対象アルゴリズム
パート4のドキュメントから、以下の検出アルゴリズムを比較対象として特定:

1. **基本的な閾値ベースアルゴリズム**
   - 統合スコアと各視点の評価スコアが閾値を超えるかどうかで判定
   - 整合性による調整を含む

2. **安定性評価を含むアルゴリズム**
   - モンテカルロシミュレーションによる安定性評価を追加
   - 入力パラメータのノイズに対する堅牢性を検証

3. **代替解生成アルゴリズム**
   - 閾値緩和、重み再検討、準静止点提示などの代替戦略を含む
   - 静止点候補が見つからない場合の対応策

## 比較軸
以下の軸で各アルゴリズムを比較:

1. **精度と信頼性**
   - 結果の正確さと信頼性
   - 誤検出や見逃しのリスク

2. **計算複雑性とパフォーマンス**
   - 計算量とリソース要件
   - 大規模データセットでのスケーラビリティ

3. **実装の複雑さ**
   - n8nでの実装難易度
   - 必要なノードとカスタムコード量

4. **柔軟性と適応性**
   - 異なるユースケースへの適応能力
   - パラメータ調整の容易さ

5. **エラー耐性**
   - 不完全または不正確なデータへの対応能力
   - エラー発生時の代替処理オプション

## 視覚表現方針
1. **比較表形式**:
   - 行: 各アルゴリズム
   - 列: 比較軸
   - セル: 評価（5段階評価や色分け）と簡潔な説明

2. **フロー図の並列表示**:
   - 各アルゴリズムの簡略化されたフロー図を並べて表示
   - 主要な違いを強調表示

3. **レーダーチャート**:
   - 各アルゴリズムの特性を多角形で表現
   - 一目で強みと弱みが分かる

4. **決定木**:
   - ユースケースや条件に基づいてアルゴリズムを選択するための決定木
   - 「もしXならアルゴリズムA、そうでなければアルゴリズムB」という形式

## n8n実装に関する注釈
各アルゴリズムのn8n実装における主要な違いを示す:

1. **基本的な閾値ベースアルゴリズム**:
   - Function, If/Switchノードが中心
   - 比較的シンプルな実装

2. **安定性評価を含むアルゴリズム**:
   - Loop, Functionノードでモンテカルロシミュレーション実装
   - より複雑なJavaScriptコードが必要

3. **代替解生成アルゴリズム**:
   - 複数の条件分岐と代替フロー
   - エラーハンドリングとフォールバック処理

## 比較図の構造
Mermaidでは以下の構造で実装:

1. 主要な比較表をクラスダイアグラムで表現
2. 簡略化されたフロー図を並べて表示
3. 決定木を表現するためのフローチャート
4. 各アルゴリズムの特性を示すレーダーチャートの視覚的表現
