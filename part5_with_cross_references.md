# コンセンサスモデルの実装（パート5：n8nによる全体オーケストレーション）

## 1. はじめに

本資料は、多視点型の意思決定支援システムの実装に関する5部構成シリーズの最終部です。このシリーズは、[パート1：基本構造と設計原則](#p1)、[パート2：基本ロジックと評価メカニズム](#p2)、[パート3：コンセンサス基準と重み付け方法](#p3)、[パート4：静止点検出と評価方法](#p4)、そして本パートで構成されています。

これまでのパートで説明した概念と技術を統合し、n8nというノーコードワークフロー自動化ツールを使用して実際のシステムとして実装する方法を解説します。[パート1](#p1)で説明した設計原則を基盤とし、[パート2](#p2)の評価メカニズム、[パート3](#p3)の重み付け方法、[パート4](#p4)の静止点検出を組み合わせた全体システムの構築方法を示します。

## 2. n8nの基本概念と設定

n8nは、ノーコードでワークフローを作成・自動化できるオープンソースツールです。多視点型意思決定支援システムの実装に適した特性を持っています。

### 2.1 n8nの概要と特徴

n8nの主な特徴は以下の通りです：

1. **ノーコード/ローコード**：プログラミングの専門知識がなくても利用可能
2. **柔軟なワークフロー設計**：複雑な条件分岐や並列処理をサポート
3. **豊富な連携機能**：多数のサービスやAPIとの連携が可能
4. **セルフホスティング**：自社サーバーでの運用が可能でデータ主権を確保
5. **拡張性**：カスタムノードの作成が可能

これらの特徴は、[パート1の設計原則](#p1-s4)、特に[柔軟性と拡張性の原則](#p1-s4-2)と[透明性と説明可能性の原則](#p1-s4-4)を実現するのに適しています。

### 2.2 n8nのインストールと初期設定

n8nのインストールと初期設定の手順：

1. **前提条件**：Node.js 14以上、npm 7以上
2. **インストール**：`npm install n8n -g`
3. **起動**：`n8n start`
4. **アクセス**：ブラウザで `http://localhost:5678` にアクセス
5. **初期設定**：ユーザー作成、APIキー設定など

詳細な設定方法は公式ドキュメントを参照してください。

### 2.3 n8nの基本操作

n8nの基本的な操作方法：

1. **ワークフロー作成**：新規ワークフローの作成
2. **ノード追加**：必要なノードの追加と設定
3. **接続設定**：ノード間の接続とデータフローの設定
4. **テスト実行**：ワークフローのテスト実行と結果確認
5. **保存と有効化**：ワークフローの保存と本番環境での有効化

これらの基本操作を習得することで、[パート1のデータフローと処理シーケンス](#p1-s5-3)を実装できるようになります。

```javascript
// Function ノードでの動的重み付け実装例
const item = $input.item;
const perspectives = item.json.perspectives;
const historicalData = item.json.historicalData;

// 動的重み付けの計算
const weights = {};
for (const perspective of perspectives) {
  const id = perspective.id;
  const baseWeight = perspective.baseWeight;
  const performanceScore = calculatePerformance(id, historicalData);
  weights[id] = baseWeight * (1 + performanceScore);
}

// 正規化
const totalWeight = Object.values(weights).reduce((sum, w) => sum + w, 0);
for (const id in weights) {
  weights[id] = weights[id] / totalWeight;
}

return {
  json: {
    weights,
    timestamp: new Date().toISOString()
  }
};
```
