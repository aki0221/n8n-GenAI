# パート1の主要概念と構造の整理

## 1. 基本構造と主要コンポーネント
- **コンセンサスモデルの4つのレイヤー**
  - 入力レイヤー: 3つの視点（テクノロジー、マーケット、ビジネス）からの情報入力
  - 評価レイヤー: 重要度評価、確信度評価、整合性評価
  - 統合レイヤー: 視点統合、静止点検出、代替解生成
  - 出力レイヤー: インサイト生成、アクション推奨、可視化

- **3つの視点の特性と役割**
  - テクノロジー視点: 基盤性（実現可能性）
  - マーケット視点: 先行性（需要・受容性）
  - ビジネス視点: 実効性（収益性・戦略適合）

## 2. 設計原則
- **視点間の関係性の尊重**: 3視点の相互関連性を考慮
- **多層的評価の実施**: 単一スコアではなく複数層での評価
- **静止点の明確な定義と検出**: 客観的基準とアルゴリズムの設定
- **透明性と説明可能性の確保**: 判断根拠の明示と追跡可能性
- **適応性と学習能力の実装**: 環境変化への対応と継続的改善

## 3. 評価メカニズム
- **重要度評価**: 影響範囲、変化の大きさ、戦略的関連性、時間的緊急性
- **確信度評価**: 情報源信頼性、データ量と質、分析手法妥当性、結果一貫性
- **整合性評価**: 視点間一致度、論理的整合性、時間的整合性、コンテキスト整合性

## 4. 実装アプローチ
- **n8nによる初期化**: パラメータ定義、ルール設定、データベーススキーマ作成
- **パラメータ設計**: 視点の重み付け、評価閾値、統合ルール
- **ルールベースロジック**: 条件式とアクションによる評価・統合ルール
