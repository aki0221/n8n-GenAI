# アーキテクチャ図作成のための留意事項

## 1. 機能的観点での留意事項

### 1.1. コンポーネント間の関係性の明確化
- **データフロー**: コンポーネント間のデータの流れを明示的に示す必要がある
  - データ収集 → 分析 → 評価 → 統合 → 出力 の主要フロー
  - 管理コンポーネントからの制御フローを区別して表現
- **依存関係**: あるコンポーネントが他のコンポーネントに依存している場合、その関係性を明示
- **双方向の通信**: 双方向のデータ交換がある場合は、それを区別して表現

### 1.2. コンポーネントの役割と責務の表現
- **コンポーネントの分類**: 処理コンポーネントと管理コンポーネントを視覚的に区別
- **サブコンポーネント**: 各主要コンポーネント内の機能単位（サブコンポーネント）を階層的に表現
- **機能の独立性**: 各コンポーネントの機能的独立性と連携ポイントを明確に示す

### 1.3. システム全体の俯瞰性
- **全体像の把握**: システム全体の構造が一目で理解できるよう配置
- **抽象化レベル**: 適切な抽象化レベルで表現し、詳細すぎず概略すぎない粒度に調整
- **拡張ポイント**: システムの拡張可能性や将来的な機能追加ポイントを示唆

## 2. 技術的観点での留意事項

### 2.1. n8nワークフローとの対応
- **ワークフロー構造**: n8nのワークフロー構造（ノードとエッジ）との対応関係を考慮
- **ノードタイプ**: 使用される主要なn8nノードタイプ（HTTP Request, Function, IF等）を反映
- **ワークフロー間連携**: 複数のワークフロー間の連携方法（Execute Workflow等）を表現

### 2.2. データ構造の表現
- **データ形式**: 各コンポーネント間で受け渡されるデータの形式（JSON等）を考慮
- **データ変換**: あるフォーマットから別のフォーマットへの変換ポイントを明示
- **データ永続化**: データベースやファイルシステムとの連携ポイントを表現

### 2.3. 外部システム連携
- **APIエンドポイント**: 外部APIとの連携ポイントを明示
- **認証メカニズム**: 認証が必要な連携ポイントを区別して表現
- **同期/非同期**: 同期処理と非同期処理を区別して表現

## 3. 描画観点での留意事項

### 3.1. レイアウトと配置
- **左から右へのフロー**: 主要なデータフローは左から右へ流れるよう配置
- **上から下への階層**: 抽象度や制御関係は上から下へ表現
- **グルーピング**: 関連するコンポーネントは視覚的にグループ化
- **バランス**: 図の全体的なバランスを考慮し、一部に偏らないよう配置

### 3.2. 視覚的表現
- **形状の使い分け**: コンポーネントの種類や役割に応じて形状を使い分け
  - 処理コンポーネント: 角丸四角形（青系）
  - 管理コンポーネント: 角丸四角形（オレンジ系）
  - サブコンポーネント: 通常の四角形（薄い色調）
- **線種の使い分け**: データフローと制御フローで線種を区別
  - データフロー: 実線矢印
  - 制御フロー: 破線矢印
- **色の使用**: 機能グループごとに色分けし、視認性を高める
  - 処理系: 青系統
  - 管理系: オレンジ/黄系統
  - 外部連携: 緑系統

### 3.3. テキストと注釈
- **フォント**: 日本語表示に適したフォント（Noto Sans JPなど）を使用
- **テキストサイズ**: 階層に応じたテキストサイズの使い分け
  - コンポーネント名: 大きめ（14-16pt）
  - サブコンポーネント名: 中程度（12-14pt）
  - 注釈: 小さめ（10-12pt）
- **ラベル配置**: 線や矢印のラベルは読みやすい位置に配置

### 3.4. 凡例と補足情報
- **凡例**: 使用している形状や色、線種の意味を説明する凡例を追加
- **バージョン情報**: 図の作成日やバージョンを記載
- **補足説明**: 複雑な関係性や特殊なフローには補足説明を追加

## 4. 実装時の具体的な注意点

### 4.1. SVG形式での実装
- **ベクター形式**: 拡大縮小しても品質が劣化しないSVG形式を使用
- **座標設定**: 要素の位置を正確に指定するための座標設定
- **グループ化**: 関連する要素はグループ化して管理
- **スタイル定義**: 再利用可能なスタイル定義を活用

### 4.2. 日本語フォント対応
- **Webフォント**: Google FontsなどのWebフォントを活用（例: Noto Sans JP）
- **フォールバック**: フォントが利用できない環境でのフォールバック指定
- **文字エンコーディング**: UTF-8エンコーディングの明示的指定
- **テキスト配置**: 日本語テキストの中央揃えや行間調整

### 4.3. アクセシビリティ
- **代替テキスト**: 図の内容を説明する代替テキストの提供
- **コントラスト**: 十分なコントラスト比の確保
- **スクリーンリーダー対応**: SVGの構造化とARIA属性の適切な使用

## 5. 本文への統合方法

### 5.1. 参照方法
- **インライン参照**: 本文中で「図1: コンセンサスモデル全体アーキテクチャ」のように参照
- **相互参照**: 図と本文の関連箇所を相互参照

### 5.2. 配置位置
- **関連セクションの直後**: 図を説明する本文の直後に配置
- **ページ内バランス**: ページ内でのバランスを考慮した配置

### 5.3. キャプションと説明
- **簡潔なキャプション**: 図の内容を端的に表すキャプション
- **詳細な説明**: 必要に応じて図の詳細を説明する補足テキスト
