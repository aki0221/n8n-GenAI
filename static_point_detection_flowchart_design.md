# 静止点検出プロセスの全体フローチャート設計

## 設計目的
静止点検出プロセスの全体像を視覚的に表現し、各ステップの関係性と処理フローを明確に示すフローチャートを作成する。

## 設計要件
1. パート4のドキュメントに記載されている静止点検出プロセスの全ステップを網羅
2. 各ステップの入出力関係を明示
3. 条件分岐と代替処理パスを明確に表現
4. n8nでの実装方法に関する注釈を含める
5. 色分けとアイコンで処理タイプを区別
6. 複雑なサブプロセスは適切に分割・グループ化

## 主要コンポーネント

### 1. データ取得・前処理
- 3つの視点（テクノロジー、マーケット、ビジネス）からの評価データ取得
- 評価スコア、確信度、重要度（重み）の取得
- データ検証と前処理（欠損値チェック、異常値検出など）

### 2. 統合スコア計算
- 各視点の評価スコア × 確信度 × 重要度の計算
- 合計による統合スコアの算出

### 3. 整合性評価と調整
- 3つの視点間の整合性スコア計算（標準偏差など）
- 整合性に基づく統合スコアの調整
- 整合性チェックと代替処理

### 4. 静止点候補チェック
- 調整後統合スコアの閾値チェック
- 各視点の個別閾値チェック
- 静止点候補の特定

### 5. 安定性評価
- 入力パラメータへのノイズ付加
- モンテカルロシミュレーションによる安定性評価
- 安定性スコアの閾値チェック

### 6. 最終的な静止点決定
- 安定性評価をクリアした候補のランク付け
- 最終的な静止点の選定
- 代替解生成（必要な場合）

### 7. 結果出力と活用
- 静止点情報の保存
- 結果の可視化
- 関連システムへの通知

## 視覚的表現方針
1. **色分け**:
   - データ取得・前処理: 青系
   - 計算処理: 緑系
   - 条件分岐: 黄系
   - 検証・評価: オレンジ系
   - 結果出力: 紫系
   - エラー・代替処理: 赤系

2. **形状**:
   - プロセス: 長方形
   - 決定: ダイヤモンド形
   - 開始/終了: 丸角の長方形
   - サブプロセス: 二重線の長方形
   - データ: 平行四辺形

3. **グループ化**:
   - 関連するプロセスをサブグラフでグループ化
   - 主要な処理ブロックに見出しを付ける

4. **注釈**:
   - n8n実装に関する注釈を追加
   - 重要なパラメータや閾値に関する説明を含める

## n8n実装に関する注釈
1. データ取得: Google Sheets Read, Postgres, HTTP Requestノード
2. 計算処理: Functionノード（JavaScript）
3. 条件分岐: If, Switchノード
4. ループ処理: Loopノード（安定性評価用）
5. データ操作: Set, Mergeノード
6. エラーハンドリング: Error Triggerノード
7. 結果出力: Postgres, Google Sheets Write, HTTP Requestノード

## フローチャートの構造
トップダウン方式で、データの流れに沿って上から下へ処理が進む構造とする。主要な条件分岐は左右に分かれ、エラーや代替処理は右側に配置する。複雑なサブプロセスは折りたたみ可能なサブグラフとして表現する。
